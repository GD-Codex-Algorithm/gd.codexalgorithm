Basic movement algorithm

A basic movement algorithm for an NPC will update the character's position to move towards a target position. The input to the algorithm will include the NPC's current position, the target position, and the speed of the NPC. The output will be the updated position of the NPC based on the desired velocity.

Pseudo Code

        

// Define the NPC's initial position
current_position: Vector2 or Vector 3

// Define the target position 
target_position: Vector2 or Vector 3

// Define the speed of the NPC 
speed = float

// Update function called every frame 
function update(): 
// Calculate the direction vector towards the target 
direction= target_position - current_position

// Normalize the direction vector
// if Vector2 is:
magnitude = √(x2 + y2)
direction.x = direction.x / magnitude 
direction.y = direction.y / magnitude 

// if Vector3 is:
magnitude = √(x2 + y2 + z2)
direction.x = direction.x / magnitude 
direction.y = direction.y / magnitude
direction.z = direction.z / magnitude

// Calculate the velocity vector 
velocity = direction * speed 

// Update the NPC's position based on velocity 
current_position += velocity

More informations

This algorithm can be used in various games where NPCs need to move towards targets. For example, in a strategy game, NPC units can move towards enemy units or objectives using this movement algorithm. In a real-time strategy game like StarCraft, NPC units can move towards enemy units or resources. The movement algorithm ensures the NPC moves in the correct direction and updates its position each frame. Games Using Similar Algorithms Many games use variations of this basic NPC movement algorithm. Some examples include: Age of Empires: NPC units move towards enemy units, resources, or waypoints. The Sims: NPC characters navigate their environment to interact with objects and other characters. This algorithm forms the foundation for more advanced NPC behaviours, such as pathfinding and obstacle avoidance.
